  namespace.yaml
  configmap.yaml

 use port-forward:
kubectl port-forward svc/monitoring-dashboard-service 8501:8501 -n ml-monitoring
Then open:
http://localhost:8501
Apply everything
kubectl apply -f k8s/namespace.yaml
kubectl apply -f k8s/configmap.yaml
kubectl apply -f k8s/secret.yaml
kubectl apply -f k8s/deployment.yaml
kubectl apply -f k8s/service.yaml
Check status:
kubectl get pods -n ml-monitoring
kubectl get svc -n ml-monitoring
kubectl logs -f deployment/monitoring-dashboard -n ml-monitoring
Main idea:
Deployment runs your dashboard container.
Service exposes Streamlit on port 8501.
ConfigMap stores normal environment variables.
Secret stores database connection URLs.


Apply in this order:
kubectl apply -f namespace.yaml
kubectl apply -f configmap.yaml
kubectl apply -f secret.yaml
kubectl apply -f deployment.yaml
kubectl apply -f service.yaml

Then test:
kubectl get pods -n ml-monitoring
kubectl logs -f deployment/monitoring-dashboard -n ml-monitoring
