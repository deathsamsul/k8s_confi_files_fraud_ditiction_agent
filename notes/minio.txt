# BUCKET JOB CREATION
minio image/container      -> runs MinIO server
minio/mc image/container   -> temporary client that creates bucket
1. MinIO StatefulSet starts
2. MinIO Service exposes it as http://minio:9000
3. Bucket Job starts using minio/mc image
4. Job connects to http://minio:9000
5. Job creates bucket mlflow
6. Job finishes
1. Server container should only run MinIO server
2. Bucket creation should happen after server is ready
3. Job can retry safely
4. Job runs once and exits
5. Cleaner for Kubernetes

MLFLOW_S3_ENDPOINT_URL: http://minio.mlops.svc.cluster.local:9000
minio = Service name
mlops = Namespace name
svc = Kubernetes service DNS zone
cluster.local = Kubernetes cluster domain
So it means:
http://<service-name>.<namespace>.svc.cluster.local:<port>
