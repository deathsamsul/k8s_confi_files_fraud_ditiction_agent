Apply order:
kubectl apply -f 00-namespace.yaml
kubectl apply -f 01-configmap.yaml
kubectl apply -f 02-db-secret.yaml
kubectl apply -f 03-dataset-pvc.yaml
kubectl apply -f 04-dataset-copy-pod.yaml

Copy CSV:
kubectl cp ./datasets/fraud_train_data.csv \
  fraud-mlops/dataset-copy-pod:/opt/datasets/fraud_train_data.csv

Run jobs:
kubectl apply -f 05-store-training-data-job.yaml
kubectl wait --for=condition=complete job/store-training-data -n fraud-mlops --timeout=600s
kubectl apply -f 06-initial-model-job.yaml

Namespace              → keeps resources organized
Secret                 → stores DB URL
PVC                    → holds CSV for first job
dataset-copy-pod       → lets you kubectl cp CSV into PVC
store-training-data    → loads CSV into PostgreSQL
initial-model          → trains/registers model from PostgreSQL
