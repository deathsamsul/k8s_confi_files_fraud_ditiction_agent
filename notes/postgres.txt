psql is the PostgreSQL command-line client.
It is used to connect to a PostgreSQL server and run SQL commands.
psql -h postgres-cluster-rw -U postgres_admin -d postgres
connect to PostgreSQL server
host = postgres-cluster-rw
user = postgres_admin
database = postgres

pg_isready -h postgres-cluster-rw -p 5432 -U postgres_admin
check if PostgreSQL server at postgres-cluster-rw:5432 is ready

CNPG PostgreSQL pod = real database server
Job postgres:17 pod = temporary client that runs psql

args = arguments passed to command
| = YAML multi-line text block


until pg_isready -h postgres-cluster-rw -p 5432 -U postgres_admin; do
  sleep 2
done
until = keep looping until command succeeds
pg_isready = check PostgreSQL readiness
-h = host
postgres-cluster-rw = CNPG read/write service
-p = port
5432 = PostgreSQL port
-U = username
postgres_admin = admin user
; = end of condition command
do = start loop body
sleep 2 = wait 2 seconds
done = end loop

Check PostgreSQL.
If not ready, wait 2 seconds.
Check again.
Repeat until ready.


psql -h postgres-cluster-rw -U postgres_admin -d postgres <<'SQL'
psql = PostgreSQL command-line client
-h = host
postgres-cluster-rw = CNPG primary database service
-U = username
postgres_admin = admin user
-d = database name
postgres = connect to database named postgres
<<'SQL' = start sending multi-line SQL into psql
Example:
psql ... <<'SQL'
CREATE DATABASE appdata;
SQL
open psql connection
run everything between <<'SQL' and SQL
close input
The ending SQL means:
stop sending SQL commands to psql
Important:
SQL is just a marker name
